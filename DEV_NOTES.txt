Coinbase Wallet Localhost Configuration

  Option 1: Quick Fix (No Developer Account Needed)

  For development/testing, you can bypass this by:

  1. Use MetaMask instead - it doesn't have this restriction
  2. Use WalletConnect - Coinbase Wallet supports WalletConnect protocol
  3. Test on a deployed domain - deploy to Vercel/Netlify for testing

  Option 2: Proper Setup (Recommended for Production)

  1. Create Coinbase Developer Account: https://developers.coinbase.com/
  2. Register your app in the Coinbase Developer Portal
  3. Add localhost domains to allowed origins:
    - http://localhost:3000
    - http://127.0.0.1:3000
    - http://localhost:3001 (if needed for backend)
  4. Get your App ID and add it to the wagmi config

  Phase 2: Popularity Scoring Algorithm

  Popularity Score Formula:
  popularityScore = (views * 1) + (purchases * 10) + (likes * 5)

  Rationale:
  - Views (1x weight): Basic engagement indicator
  - Purchases (10x weight): Strong intent signal - someone paid for it
  - Likes (5x weight): Social validation, easier than purchase but more than view

  Alternative time-decay formula:
  // Boost recent content
  const daysSincePublish = (Date.now() - publishDate) / (1000 * 60 * 60 * 24);
  const timeFactor = Math.max(0.1, 1 - (daysSincePublish / 30)); // Decay over 30 days
  popularityScore = baseScore * timeFactor;
